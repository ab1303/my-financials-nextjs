```sql
ALTER TABLE "ImportSession" ADD COLUMN "startDate" TIMESTAMP(3);
ALTER TABLE "ImportSession" ADD COLUMN "endDate"   TIMESTAMP(3);
```

Both columns nullable — safe to add with no data migration.

## Phase 2 — CSV Confirm Route

### Location
`src/app/api/transactions/csv/confirm/route.ts`

### Change

After both `confirmDebitTransactions` and `confirmCreditTransactions` resolve, aggregate the date range **before** the `importSession.update` call:

```typescript
// Aggregate min/max transaction date for this import session
const dateRange = await prisma.transaction.aggregate({
  where: { importSessionId: fileId },
  _min: { date: true },
  _max: { date: true },
});

await prisma.importSession.update({
  where: { id: fileId },
  data: {
    status,
    recordsCreated: totalEntries,
    startDate: dateRange._min.date ?? null,
    endDate:   dateRange._max.date ?? null,
  },
});
```

No other changes to the route. The aggregate is a single indexed query (`importSessionId` has an index via `@@index([importSessionId])` on `Transaction`).

### Test Cases

| Test | Type | Verifies |
|---|---|---|
| POST /confirm with valid debit+credit months → `ImportSession.startDate` = min transaction date | Integration | Date range written correctly |
| POST /confirm with all transactions on the same date → `startDate === endDate` | Integration | Same-day edge case handled |
| POST /confirm where all transactions are EXCLUDED (0 `totalEntries`) → `startDate` / `endDate` are null | Integration | Empty import doesn't fail aggregate |
| POST /confirm with FAILED status (0 transactions saved) → `startDate` / `endDate` null | Integration | Aggregate on empty set returns null safely |

```typescript
function formatCoverage(startDate: string | null, endDate: string | null): string {
  if (!startDate || !endDate) return '—';

  const opts: Intl.DateTimeFormatOptions = { day: 'numeric', month: 'short', year: 'numeric' };
  const start = new Date(startDate).toLocaleDateString('en-AU', opts);
  const end   = new Date(endDate).toLocaleDateString('en-AU', opts);

  return start === end ? start : `${start} – ${end}`;
}
```

## Edge Cases Summary

| Case | Behaviour |
|---|---|
| PENDING session (no transactions yet) | `startDate`/`endDate` = null → "—" |
| Legacy session (pre-feature) | `startDate`/`endDate` = null → "—" |
| All transactions excluded (`totalEntries = 0`) | Aggregate returns null → "—" |
| All transactions on same date | `startDate === endDate` → single date displayed |
| FAILED import | `startDate`/`endDate` = null (no transactions written) → "—" |
| AI import with transaction dates | Populated same as CSV |
| Timezone edge case | Dates stored as UTC; `toLocaleDateString('en-AU')` renders in browser locale (acceptable) |

## Integration Points

- `csv/confirm/route.ts` already holds `fileId` (the `ImportSession.id`) and has access to `prisma` — no new parameters needed
- `prisma.transaction.aggregate` is available without extra Prisma config
- `listImportSessions` already maps `s` fields — adding two more mapped fields is non-breaking
- `ImportSessionHistory.tsx` `SessionRow` is a local interface — updating it triggers TypeScript compile check across the component
